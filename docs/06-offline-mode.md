# 오프라인 모드

Holonet은 백엔드 서버 없이도 완전히 동작할 수 있습니다. 모든 데이터는 로컬에 저장되며, 서버가 다시 연결되면 자동으로 동기화됩니다.

## 🔌 오프라인 모드란?

오프라인 모드는 백엔드 서버가 실행되지 않아도 Holonet 앱이 완전히 동작하는 기능입니다.

### 주요 특징

- ✅ **서버 없이 동작**: 백엔드 서버가 없어도 모든 기능 사용 가능
- ✅ **자동 폴백**: 서버 연결 실패 시 자동으로 로컬 모드로 전환
- ✅ **로컬 저장**: 모든 데이터가 IndexedDB에 저장
- ✅ **투명한 동작**: 사용자는 온라인/오프라인을 구분할 필요 없음

## 💾 데이터 저장 위치

### IndexedDB

모든 데이터는 브라우저의 IndexedDB에 저장됩니다:
- **Workspaces**: 워크스페이스 목록
- **Items**: API 요청 및 폴더
- **Environments**: 환경 변수
- **Sync State**: 동기화 상태

### 저장소 구조

```
IndexedDB (holonet)
├── workspaces
├── items
├── environments
└── syncState
```

## 🔄 동작 방식

### 앱 시작 시

1. **서버 연결 시도**
   - 백엔드 서버에 연결을 시도합니다
   - 연결 성공: 서버 데이터와 로컬 데이터를 병합
   - 연결 실패: 로컬 데이터만 사용

2. **상태 표시**
   - 콘솔에 "Server is available" 또는 "Server is not available" 메시지 표시

### 데이터 읽기

- **온라인**: 서버 데이터 우선, 없으면 로컬 데이터
- **오프라인**: 로컬 데이터만 사용

### 데이터 쓰기

- **온라인**: 서버에 저장 + 로컬에 백업
- **오프라인**: 로컬에만 저장

### 자동 폴백

서버 요청이 실패하면:
1. 자동으로 로컬 저장소로 전환
2. 로컬에 데이터 저장
3. 사용자에게 투명하게 동작

## 📝 사용 예시

### 예시 1: 서버 없이 워크스페이스 생성

1. 백엔드 서버를 실행하지 않습니다
2. 앱을 실행합니다
3. 워크스페이스를 생성합니다
4. ✅ 정상적으로 생성됩니다 (로컬에 저장)

### 예시 2: 오프라인에서 API 요청 관리

1. 오프라인 상태에서도:
   - 워크스페이스 생성/수정/삭제 가능
   - API 요청 추가/수정/삭제 가능
   - 환경 변수 관리 가능

### 예시 3: 온라인 복귀

1. 서버를 다시 시작합니다
2. 앱을 다시 실행합니다
3. 서버 연결이 성공하면:
   - 로컬 데이터와 서버 데이터가 병합됩니다
   - 향후 버전에서 자동 동기화가 지원될 예정입니다

## 🔍 오프라인 모드 확인

### 콘솔 메시지

개발자 도구 콘솔에서 확인:
- `"Server is available"` - 온라인 모드
- `"Server is not available, using offline mode"` - 오프라인 모드

### 동작 확인

서버 없이도 다음이 정상 동작하면 오프라인 모드가 작동 중입니다:
- 워크스페이스 생성
- API 요청 추가
- 데이터 조회

## ⚠️ 제한사항

### 현재 제한

- **실시간 동기화**: 다른 클라이언트와의 실시간 동기화는 서버가 필요합니다
- **서버 동기화**: 오프라인에서 변경한 데이터는 향후 버전에서 자동 동기화될 예정입니다

### 데이터 영구성

- IndexedDB 데이터는 브라우저를 닫아도 유지됩니다
- 브라우저 데이터를 삭제하면 로컬 데이터도 삭제됩니다
